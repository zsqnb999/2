# P2053 原代码问题定位（不是重写版）

你给的那份代码核心是最小费用最大流建模，思路本身是对的：

- 源点 -> 每辆车（容量 1，费用 0）
- 车 -> 技师 j 的第 k 个服务位（容量 1，费用 `t[i][j] * k`）
- 服务位 -> 汇点（容量 1，费用 0）

但是实现里有几个关键错误会导致结果不稳定甚至越界。

---

## 1) `dis[] / cur[] / vis` 的大小与图节点总数不匹配

原代码定义：

- `constexpr int N=65, M=10;`
- `ll dis[N], cur[N];`
- `bitset<N> vis;`

但真实节点编号上限是：

- `S = n + n*m + 1`
- `T = S + 1 = n + n*m + 2`

当 `n=60, m=9` 时：

- `T = 60 + 540 + 2 = 602`

远大于 65。于是 `dis[T]`、`vis[T]`、`cur[T]` 都会越界访问，行为未定义。

---

## 2) SPFA 初始化范围错误（只初始化到 `n`）

原代码：

```cpp
fill(dis, dis+1+n, 1e18);
```

这只把 `dis[0..n]` 初始化了，而图中大量节点（服务位、S、T）都在 `n` 之后。会导致：

- 大量节点距离保留旧值（脏数据）
- SPFA 松弛过程在错误初值下运行
- 最短路判定可能错误

正确做法是初始化到图中实际最大节点（至少到 `T`）。

---

## 3) 最终输出变量用错：输出了流量平均，不是费用平均

原代码：

```cpp
ll ans=0;
while(spfa()){
    ll f=0;
    while(f=dfs(S,T,1e18)) ans+=f;
}
double rs=ans;
printf("%.2lf", rs/n/1.0);
```

这里 `ans` 累加的是 `f`（增广流量），而每次单位容量增广时总流量最后就是 `n`，所以 `ans/n` 近似恒为 `1.00`（或非常接近固定值），这明显不是题目要的平均等待时间。

真正的总费用在你代码里是 `ret`（`ret += x * edge_cost`），应该输出 `ret / n`。

---

## 4) 建图数组上限也偏紧/不可维护

`vector<node> ps[N*M+5];` 在常量下是 655，虽然对 `n<=60,m<=9` 勉强够，但其余结构没同步，容易“某些数组够大、某些不够大”的错配。

建议统一以实际点数 `V` 申请所有图相关容器（或统一常量并严格覆盖 `T`）。

---

## 结论

你的思路（把每个技师拆成按顺序的 `k=1..n` 服务位）是对的，问题主要在实现细节：

1. 节点空间管理错误（越界）
2. SPFA 距离初始化范围错误
3. 输出用了流量而非费用

这三个问题里，**第 3 个会直接导致答案语义错**，**前两个会导致结果不稳定甚至 RE/WA**。
